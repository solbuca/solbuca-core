## Решения (принято на текущий момент)

- **Мобайл:** React Native (Solana App Kit), один кодовый репозиторий на iOS + Android.
- **Кошелёк / онбординг:** Privy embedded wallet + social login (Apple/Google), тихое создание кошелька (MPC, без seed-фразы). MWA как fallback для крипто-нативных.
- **On-chain (Anchor / Rust):**
  - `loyalty` — баллы как PDA (в MVP НЕ торгуемый токен, чтобы не попасть под секьюрити-регуляцию).
  - `membership` — NFT-членство баров («злые логотипы»).
  - `payments` — Solana Pay чекаут в баре.
  - `tournament` — escrow призового пула, skill-based (фаза 2).
- **Бэкенд:** API gateway + индексер (on-chain → БД) + Postgres + хранилище медиа.
- **Оплата в стейблкоине (USDC), не в SOL** — чтобы цена не скакала между сканом и подтверждением.

## Фазы

- **0 — каркас:** монорепо, App Kit-шаблон на обеих платформах, онбординг на devnet, пустые программы деплоятся.
- **1 — грантовый MVP:** один бар, `membership` + `payments` + `loyalty` (заработал/потратил). Цель — заявка на Solana Foundation Grant (категория Payments).
- **2 — контент:** лента рецептов + видео, голосование (skill-based, без ставок), `tournament`.
- **3 — монетизация:** торгуемый токен / стейкинг / вывод в фиат. Только после юридической проработки.

## Стратегия по деньгам

- Dev-тулинг (snapshot CLI для тестов Solana-программ) → **Solana Foundation Grant** (чистый фит в Developer Tooling).
- Коктейльная платформа целиком → **Colosseum / хакатон / seed** (потребительский продукт, не Foundation-грант).
- Скрытое преимущество: автор — security researcher → аудит-устойчивые контракты с первого дня (дифференциатор для гранта и для доверия баров).

## Записная книжка (идеи на потом, думаем)

### Реферальная программа для подключения баров
- % заработка с подключённых баров — рефереру за приведённый бар.
- Источник дохода под вопросом: комиссия платформы ИЛИ оплата от баров за рекламу/продвижение.
- Рефералы по барам (трекинг, кто кого привёл).
- **Зачем это правильно:** подключение баров (BD) — главный бутылочный горлышко проекта; реферальная схема атакует именно его.
- **Проверить при проработке:** юридическое оформление rev-share (выплаты рефереру), как это стыкуется с skill-vs-gambling рамкой и с моделью комиссий.
- Статус: **parked, думаем.**

### Карта и геолокация заведений (бары, клубы на контракте)
- Карта с маркерами заведений-партнёров; маркер = membership-NFT бара («злой логотип» как пин на карте).
- Гео заведений хранится off-chain (Postgres: lat/long, статус «на контракте»).
- **MVP-релевантно:** базовый каркас карты в Home нужен уже в фазе 1 (даже с одним баром); полную ценность даёт при многих заведениях.
- Стек: react-native-maps / Mapbox.

### Proof-of-visit + ранги (геймификация, фаза 2)
- После посещения + оплаты — отметка о визите, возможно как NFT (POAP-стиль).
- На Solana дёшево через **compressed NFT (cNFT)** — доли цента за минт, поэтому отметка за каждый визит экономически ок.
- Коллекция визитов по разным заведениям → «коллекционер»; частота → «постоянный посетитель»; и другие ранги.
- Ранги считаются из on-chain активности (число визитов, собранные заведения, траты) — это слой `reputation`, отдельно хранить не нужно.
- **Проверить при проработке:**
  - Чем доказывается реальный визит? Привязывать к ОПЛАТЕ (самый честный proof, визит-NFT как побочный продукт транзакции) или к QR заведения — НЕ к голому GPS (спуфится).
  - Apple App Store, правило 3.1.1: NFT не должны разблокировать функции в приложении (иначе требует IAP с комиссией). Перки за ранги формулировать как РЕАЛЬНЫЕ награды в баре (бесплатный коктейль, проход без очереди), не как цифровые разблокировки в приложении.
- Статус: **parked, фаза 2.**

### Бронирование, столы, предзаказы, спецзаказы (фаза 2–3)
- Бронь столов в барах/клубах + размещение столов (схема зала заведения, выбор конкретного стола/зоны).
- Предзаказы: напитки/бутылки заранее, предоплата через приложение (особенно bottle service для клубов + VIP-стол).
- Спецзаказы: кастомные запросы (особый коктейль, конкретная бутылка, день рождения) — концьерж-функция.
- **On-chain / off-chain:** почти всё off-chain (доступность, слоты, схема зала, инвентарь столов — Postgres). On-chain ТОЛЬКО денежный поток: депозит за бронь (USDC escrow) и предоплата предзаказа — и то и другое уже идёт через `payments` + даёт баллы и proof-of-visit. Контрактный surface остаётся маленьким (плюс для безопасности).
- **Депозит-бронь как аргумент для баров (BD!):** предоплата убивает no-show — реальная боль заведений. Депозит применяется к счёту или сгорает по политике. Сильный pitch при подключении баров; для клубов — предзаказ бутылки + VIP-стол = высокий гарантированный чек.
- **Проверить:** политика сгорания депозита (защита прав потребителя / возвраты) — юридический нюанс.
- **Scope-флаг:** бронь — это целый продукт сам по себе (уровень Resy/OpenTable). Требует мерчант-сайд дашборд для бара (управление столами, календарь броней, меню предзаказов) — отдельная бар-фейсинг поверхность. НЕ в MVP.
- Статус: **parked, фаза 2–3.**

### Меню коктейлей (кликабельное) — MVP-релевантно
- Простое кликабельное меню заведения (коктейли, цена, описание).
- Нужно для заказа/предзаказа и как контент; теги алкоголя используются для внутреннего harm-reduction сигнала (не показываемого).
- **Каждый пункт меню тегируется алкогольным содержанием** (стандартные дринки / граммы спирта) + флаг «безалкогольный». Нужно для внутреннего сигнала «перебора» (harm-reduction) и фильтра безалкогольных опций. Разовая разметка на стороне бара.

### Промилле — НЕ показывать; вместо этого мягкая геймификация (фаза 2)
- **Решение: счётчик/цифры промилле НЕ показываем юзеру** (убирает риск «решил, что можно за руль»).
- Внутренний сигнал «перебора» можно использовать — но НЕ для награды за перебор, а для harm-reduction.
- **Геймификация / ранги:** «король вечеринок», «королева бара/клуба» и т.п., бонусы от нас и от бара.
- **Главный принцип дизайна: награждать ЛОЯЛЬНОСТЬ и патронаж, а не объём выпитого.**
  - Ранги по: визитам, частоте, стрику, разнообразию заказов, «прошёл всё меню» (collector-связка), приведённым друзьям (реферал-связка), кумулятивному спенду за сезон (VIP-тиры как авиамили). НЕ «кто больше выпил за вечер».
  - Момент «перебора» → не приз за перебор, а награда за ОТВЕТСТВЕННЫЙ ход: раунд воды, безалкогольный коктейль, доехал на такси. Те же NFT/бейджи/скидки, вектор в правильную сторону. Уникальный PR-нарратив «приложение, которое заботится».
  - «Похмельный коктейль» со скидкой = retention-хук («вернись завтра»), НО recovery-дринк (бранч/безалкогольный/электролиты), не hair-of-the-dog.
- **Флаг:** правила ответственного маркетинга алкоголя + лицензии баров — поощрение чрезмерного потребления может быть юридической/лицензионной проблемой. Вектор «лояльность, не объём» = ещё и защита.
- Статус: **parked, фаза 2.**
